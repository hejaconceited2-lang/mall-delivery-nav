# MallNav · 商城配送导航系统

办公楼宇内部送餐的店铺定位工具。基于消防疏散图，OCR提取店铺编号并标注坐标，配送员输入编号即可在地图上定位。

## 线上地址

**https://hejaconceited2-lang.github.io/mall-delivery-nav/**

## 技术架构

纯前端单页应用，无需服务器。数据存储在浏览器 localStorage 中。

```
nav_system.html          # 主应用（搜索 + 楼层图定位 + 数据管理）
floor_maps_viewer.html   # 疏散图裸图查看器（备用）
index.html               # 首页重定向
*.jpg                    # 4张楼层消防疏散图
```

## 楼层覆盖

| 楼层 | 图片 | 编号格式 | 预提取数量 | 状态 |
|------|------|----------|------------|------|
| 1F | a6bbce... | 1xxx | 0 | OCR困难，待人工补录 |
| 2F | 9b534d... | 2xxx~22xx | 22 | 已OCR提取，待人工校准 |
| 3F | 679d8c... | 30xx~31xx | 19 | 已导入 |
| 4F | 132602... | 40xx~41xx | 45 | 已导入 |
| 5F | 0d667d... | 5+字母+数字 | 4 | OCR困难，待人工补录 |
| 6F | 10f27b... | M-柜+区域编号 | 0 | OCR困难，待人工补录 |
| B1 | — | — | 0 | 图片缺失 |

## 数据流

1. **OCR提取** → 消防疏散图 → Tesseract OCR → 预处理去噪 → 编号+坐标JSON
2. **人工校准** → 在楼层图上点击纠正偏移的坐标
3. **搜索定位** → 输入编号 → 切换到对应楼层 → 红圈标记位置

## 开发记录

### OCR提取过程
- 引擎：Tesseract OCR + Python (Pillow, pytesseract)
- 预处理：灰度 → 对比度增强 → 锐化 → 二值化
- 3F/4F 编号规则整齐（数字连续），提取成功率 ~70%
- 5F/6F 编号含字母+色块分区，标准OCR失效
- 5F 特殊处理：S→5 字符矫正（`SA16` → `5A16`）

### 数据字段
```json
{
  "floor": "4F",
  "num": "4058",
  "elevator": "6号客梯",
  "direction": "出电梯左转，直行20米，第一个路口右转",
  "coordX": 60.2,
  "coordY": 58.3
}
```

## 使用方式

- **配送员**：打开URL → 搜索编号 → 查看楼层+走法 → 点「查看位置」
- **管理员**：数据管理Tab → 录入/编辑 → 在地图上点击标注坐标
- **数据迁移**：导出JSON备份 → 新设备导入
